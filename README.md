# PodReader

播客精读与高质量中文写作 Skill。

把播客转录稿、长文、报告、书籍章节、视频转录稿等长内容，提炼成一篇可直接阅读的中文精读文章；按需再输出为 EPUB 电子书或微信公众号 HTML。

## 重要质量预期

- 这是一个“精读提炼” Skill，不是逐字稿整理器。
- 默认目标是用更短的篇幅保留原始内容里最值得读的部分，而不是覆盖每一句话。
- 输出强调可读性、信息密度和中文写作质量；如果你要公开发布，仍建议再做一轮人工校对。
- 当原始内容信息密度偏低时，文章会主动缩短，不会为了凑字数把噪音也写进去。

## 这个 Skill 能做什么

- 从长内容中提取真正有价值的“宝石”，过滤噪音
- 自动处理专业术语：保留英文原文，用自然中文解释
- 清理口语引语，保留说话者的真实语气
- 支持单人独白、多人对谈、文章、报告、书籍章节等输入
- 内置反“AI 味”写作规范，避免套路化总结口吻
- 一键输出 EPUB 电子书（兼容微信读书等阅读器）
- 一键输出微信公众号格式（复制粘贴即保留基本排版）

## 安装

### 方式 A：上传 `.skill` 文件

下载 `podreader.skill`，在 Claude 设置页的 Skills 板块上传安装。

### 方式 B：手动放入 skills 目录

将 `SKILL.md` 复制到你自己的 skills 目录中即可。

如果你维护的是自己的技能仓库，建议把 `SKILL.md` 作为事实源文件，`podreader.skill` 作为给终端外用户的一键安装包。

## 推荐使用方式

安装后可直接在对话里说：

| 你说的话 | 触发模式 |
| --- | --- |
| “帮我精读/总结这篇内容” | 纯文字精读 |
| “做成 EPUB” | 精读 + EPUB 电子书 |
| “转公众号格式” | 精读 + 微信公众号 HTML |
| “EPUB 和公众号都要” | 同时输出两种格式 |

更稳的说法是把目标也说清楚，例如：

```text
帮我把这篇播客转录稿做成一篇 5-8 分钟能读完的中文精读文章，术语第一次出现时解释清楚，保留值得引用的原话，最后输出 EPUB。
```

如果输入是播客转录稿，建议一并提供这些元信息：

- 播客名
- 本期标题
- 嘉宾
- 期数
- 时长
- 发布日期

这些信息会优先写入 EPUB 或公众号版本的头部元信息区域。

## 写作原则

这个 Skill 的方法论脱胎于长期的播客精读实践，核心原则是：

1. **80/20 法则**：用 20% 的阅读时间拿到 80% 的价值。
2. **拦路虎先行**：读者会卡住的术语，必须尽早解释清楚。
3. **宝石驱动**：只保留真正新鲜、具体、值得讲的内容。
4. **反 AI 味**：严格限制套路句式，追求自然、有人味儿的中文。

## 输出边界

- 纯文字模式：直接在对话中输出精读文章。
- EPUB 模式：生成完整 EPUB 3.0 结构，适合导入阅读器。
- 公众号模式：生成可复制粘贴到微信公众号编辑器的 HTML。
- 如果用户没有要求特定格式，只执行精读写作层，不额外生成文件。
